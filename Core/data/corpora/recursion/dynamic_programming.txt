Dynamic programming is an algorithm design technique for solving optimization and counting problems that have a particular recursive structure. Like divide and conquer, dynamic programming decomposes a problem into smaller subproblems. But where divide-and-conquer is most effective when subproblems are independent — when solving one does not affect the solution of another — dynamic programming is designed for problems with overlapping subproblems: cases in which the same subproblem appears in many different branches of the recursion tree and would be recomputed many times if handled naively. By storing the solution to each subproblem the first time it is solved and looking it up rather than recomputing it in subsequent encounters, dynamic programming converts exponential-time naive recursion into polynomial-time computation.

The term was coined by Richard Bellman in the 1950s in a somewhat obscure sense having more to do with the mathematics of sequential decision processes than with the specific algorithmic technique now universally associated with it. Bellman's principle of optimality, however, is the key theoretical foundation: an optimal solution to any instance of a problem must consist of optimal solutions to its subproblems. This property, called optimal substructure, is what makes it valid to build up an optimal solution from optimal solutions to subproblems, and it is the essential precondition for dynamic programming to be applicable.

The Fibonacci sequence provides the simplest illustration of why memoization is valuable. The naive recursive computation of the nth Fibonacci number has exponential time complexity because each call to F(n) spawns calls to F(n-1) and F(n-2), and the resulting computation tree grows exponentially, with F(2) computed O(Fibonacci(n)) times — itself an exponential number. Memoization stores each computed Fibonacci number in a table the first time it is computed. On any subsequent call with the same argument, the stored value is returned immediately without further recursion. With memoization, each subproblem is solved exactly once, and the total number of unique subproblems is just n, so the memoized computation runs in O(n) time.

Memoization is the top-down approach to dynamic programming: one writes the recursive algorithm naturally and adds a cache. Bottom-up dynamic programming inverts this approach: one fills in a table of subproblem solutions in an order that ensures all dependencies of each subproblem have been solved before that subproblem is tackled. For the Fibonacci sequence, bottom-up dynamic programming simply fills a table from F(0) and F(1) up to F(n), each entry computed in constant time from the two preceding entries. The two approaches — top-down with memoization and bottom-up tabulation — have the same asymptotic complexity but differ in their constant factors, their handling of stack overflow, and their clarity for different types of problems.

The classic example that exhibits the power of dynamic programming most dramatically is the computation of the edit distance between two strings, also known as the Levenshtein distance. The edit distance between two strings is the minimum number of single-character insertions, deletions, or substitutions needed to transform one string into the other. A recursive characterization follows naturally: the edit distance between strings A of length m and B of length n is either the edit distance between A without its last character and B without its last character, if those characters are equal, or one plus the minimum of three edit distances corresponding to the three edit operations. This recursion has exponentially many paths if computed naively but only O(mn) unique subproblems. The bottom-up dynamic programming table fills an m-by-n matrix, computing each entry in constant time from its three neighbors, for total O(mn) time and space.

Optimal substructure is the key property that makes dynamic programming correct. It says that an optimal solution to the full problem is composed of optimal solutions to subproblems. For the shortest path problem, optimal substructure holds because any subpath of a shortest path is itself a shortest path between its endpoints. For the knapsack problem, optimal substructure holds in a particular sense: the optimal solution to packing a knapsack of capacity W with items from a set S either includes or excludes item i, and in either case the remaining items must form an optimal solution to the remaining subproblem. If optimal substructure fails — as it does for the longest simple path problem, for instance — dynamic programming does not apply directly.

Overlapping subproblems is the other essential property. Many recursively structured problems do not have overlapping subproblems — the Towers of Hanoi, for example, produces a recursion tree with no repeated subproblems — and for these, dynamic programming provides no advantage over straightforward recursion. The classic dynamic programming problems all have overlapping subproblems: Fibonacci numbers, edit distance, longest common subsequence, matrix chain multiplication, optimal binary search trees, and the various shortest-path algorithms like Dijkstra's and Floyd-Warshall all derive their efficiency from storing and reusing solutions to subproblems that appear many times in the recursion.

The design of a dynamic programming algorithm requires three steps: identifying the structure of an optimal solution (establishing optimal substructure), recursively defining the value of an optimal solution in terms of smaller subproblems, and computing the value of an optimal solution either top-down with memoization or bottom-up by filling a table. An additional step, often required in practice, is recovering an actual optimal solution rather than just its value by tracing back through the table to reconstruct the choices that led to the optimal value. The connection between dynamic programming and Bellman's principle of optimality in sequential decision problems — where at each stage one chooses an action and the optimal policy is defined by optimal future policies from each possible resulting state — is deep and has been elaborated in the theory of Markov decision processes and reinforcement learning.
