Error correction codes are methods for adding structured redundancy to data so that errors introduced during transmission or storage can be detected and corrected. The fundamental idea is counterintuitive: by adding more bits to a message, we can actually transmit it more reliably over a noisy channel. Shannon's channel coding theorem guarantees that this is possible up to the channel capacity, and the design of efficient error-correcting codes that approach this limit has been one of the central engineering challenges in communications since 1948.

The simplest error correction scheme is the repetition code: to send a single bit, transmit it three or more times. If the majority of received bits agree, use that as the decoded value. A bit sent three times over a channel that flips each bit independently with probability p will be decoded correctly with probability one minus 3p squared plus 2p cubed, which for small p is considerably better than p. However, the repetition code is very wasteful: to transmit one bit of information, three bits of channel capacity are consumed, giving a rate of one third. Better codes use redundancy more cleverly to achieve high reliability at higher rates.

Richard Hamming invented the first systematic family of efficient error-correcting codes in 1950, motivated by the unreliability of relay computing machines at Bell Labs that would reset and lose hours of computation due to single-bit errors. A Hamming code adds a small number of parity check bits to a message so that any single-bit error can be identified and corrected. In the simplest Hamming code, a message of four bits is extended to seven bits by adding three parity bits. Each parity bit is the modulo-2 sum of a specific subset of the message bits, chosen so that any single-bit error produces a unique pattern of violated parity checks. Because there are three parity bits and three violated bits, the violated checks form a three-bit binary number that gives the position of the error. The code can therefore correct any single error and detect (but not correct) any double error.

Hamming codes are examples of linear codes, a broad and important family. A linear code is one in which the set of valid codewords forms a linear subspace of the space of all binary strings of the codeword length. Linear codes are described by a generator matrix, which maps k-bit messages to n-bit codewords, and a parity check matrix H, whose rows are the parity check equations. The minimum distance of a code — the minimum number of bit positions in which any two codewords differ — determines its error-correcting capability. A code with minimum distance d can correct any combination of up to floor((d minus 1) over 2) errors.

Reed-Solomon codes, invented by Irving Reed and Gustave Solomon in 1960, are a family of non-binary linear codes with exceptional error-correcting properties. They are defined over finite fields and encode messages as polynomials: a k-symbol message is treated as the coefficients of a polynomial of degree k minus 1, and the codeword consists of the values of this polynomial evaluated at n distinct points. Because a polynomial of degree k minus 1 is determined by any k of its values, and any two distinct polynomials of degree k minus 1 can agree on at most k minus 1 points, the minimum distance of a Reed-Solomon code is n minus k plus 1. This achieves the Singleton bound, the maximum possible minimum distance for a linear code of given length and dimension.

Reed-Solomon codes are capable of correcting erasures as well as errors. An erasure occurs when the receiver knows that a symbol is incorrect but does not know what the correct value is — for instance, a data storage medium that has a scratched sector can flag that sector as erased. Correcting erasures is easier than correcting errors because the location of the problem is already known. A Reed-Solomon code can correct up to n minus k erasures or up to (n minus k) over 2 errors, or any combination where the number of errors times 2 plus the number of erasures is at most n minus k. This flexibility makes Reed-Solomon codes ideal for storage applications such as CDs, DVDs, and Blu-ray discs, where physical scratches cause erasures, and for deep-space communication systems including the Voyager spacecraft.

Turbo codes, independently invented by Berrou, Glavieux, and Thitimajshima in 1993, were the first practical codes demonstrated to achieve performance within a fraction of a decibel of the Shannon limit. A turbo code uses two recursive systematic convolutional encoders applied to the same information bits but in a different order (using an interleaver), producing two streams of redundant bits. Decoding uses an iterative algorithm that passes "soft" probability information back and forth between two component decoders, each one using the output of the other as side information and refining its estimates with each iteration. This iterative, belief-propagation-style decoding is computationally feasible and converges to near-optimal performance after a moderate number of iterations. Turbo codes transformed mobile communications and are used in 3G and 4G cellular standards.

Low-density parity-check codes, or LDPC codes, were originally invented by Robert Gallager in his 1960 doctoral thesis but lay dormant until they were rediscovered and recognized as achieving near-capacity performance in the 1990s. An LDPC code is a linear code defined by a sparse parity check matrix — one with very few ones per row and column. The sparsity allows efficient iterative decoding using the sum-product algorithm, also called belief propagation, on the bipartite Tanner graph of the code. LDPC codes are now used in 5G wireless, Wi-Fi (IEEE 802.11), and digital video broadcasting because they can be designed to approach the Shannon limit for a wide variety of channels and block lengths.

The history of error-correcting codes illustrates the deep connection between information theory and practical engineering. Shannon's theorem guaranteed that reliable communication is possible, but gave no hint of how to construct the codes that achieve it. The fifty-year journey from Hamming codes to turbo codes and LDPC codes has produced both deep mathematical insights about the structure of codes and highly efficient algorithms that bring theory into everyday communication technology.
