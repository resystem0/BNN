Data compression is the process of encoding information using fewer bits than the original representation. It is one of the most practically important applications of information theory and underlies virtually all digital media storage and transmission: compressed images in JPEG format, compressed audio in MP3 files, compressed video in H.264, compressed text in zip files. The theoretical foundation for lossless compression — compression from which the original data can be perfectly recovered — is Shannon's source coding theorem, which establishes the entropy of the source as the minimum achievable compression rate.

Lossless compression is possible whenever the source data has structure or redundancy. A text file in English has enormous redundancy: the letter E appears far more frequently than the letter Q, the sequence TH is far more common than XP, and after the letters Q and U appear together, a vowel almost always follows. Any representation that assigns shorter bit strings to more common patterns and longer bit strings to rare patterns will require fewer bits on average than a fixed-length encoding. The fundamental limit on how short the compressed representation can be is the Shannon entropy of the source. A source producing symbols with entropy H bits per symbol cannot be compressed, on average, to fewer than H bits per symbol without losing information.

Huffman coding, developed by David Huffman in 1952, is an optimal prefix-free code for a source with known symbol probabilities. A prefix-free code is one in which no codeword is a prefix of another, which allows unique decoding without any separator characters. Huffman's algorithm constructs the optimal code by repeatedly combining the two least probable symbols into a single combined symbol with probability equal to their sum, building a binary tree bottom-up. The resulting codewords are read off as the path lengths from the root of the tree to each leaf. Symbols with higher probability get shorter codewords and symbols with lower probability get longer codewords, mirroring the information content of each symbol.

The average code length of a Huffman code satisfies H(X) is less than or equal to L-bar (average code length) is less than H(X) plus one, where H(X) is the entropy of the source distribution. This one-bit gap can be significant if the source entropy is small or if symbols are encoded individually. For sources with low entropy per symbol, the inefficiency of encoding one symbol at a time can be substantial. The solution is to encode longer sequences of symbols together, treating n-symbol blocks as single super-symbols. As block length increases, the average code length per original symbol approaches the entropy, closing the gap.

Arithmetic coding is a technique that achieves compression rates extremely close to the entropy even for individual symbols, effectively overcoming the integer-length constraint that limits Huffman coding. Instead of assigning a fixed-length codeword to each symbol, arithmetic coding assigns to an entire message a single fraction in the interval between zero and one, subdividing the interval according to symbol probabilities with each new symbol. The final interval's boundaries contain all the information needed to reconstruct the original sequence, and the number of bits needed to specify a point in the interval approaches the entropy of the sequence. Arithmetic coding is used in the deflate algorithm underlying the zip format, and in many image and video codecs.

Universal compression algorithms are those that approach the optimal compression rate for a broad class of sources without requiring prior knowledge of the source statistics. The Lempel-Ziv family of algorithms, developed by Abraham Lempel and Jacob Ziv in 1977 and 1978, achieves asymptotically optimal compression for any stationary ergodic source. The LZ77 algorithm encodes data as references to previously seen patterns in a sliding window: instead of literally encoding a repeated string, it encodes a pointer to where that string appeared earlier in the data and how long it is. The LZ78 algorithm builds an explicit dictionary of recurring patterns incrementally as it processes the data. Variants of these algorithms — LZW, LZSS, and others — are the basis of gzip, the GIF image format, and many other widespread compression tools.

Dictionary-based methods like Lempel-Ziv exploit statistical structure across long spans of the input. Context modeling methods are an alternative approach that explicitly estimate the probability distribution of each symbol given its context — the preceding symbols — and use that estimated distribution to drive an arithmetic coder. The prediction by partial matching, or PPM, algorithm builds context models of varying orders and blends their predictions. Modern compressors like bzip2 and 7-zip incorporate Burrows-Wheeler transform preprocessing, which reorders the data to concentrate repetitions together, followed by move-to-front encoding and run-length encoding to exploit these repetitions before applying entropy coding.

The entropy bound cannot be violated by lossless compression, but different models of the source give different entropy estimates, and better modeling leads to better compression. An ideal compressor that knew the true probability model of the source could achieve exactly the entropy rate. In practice, the challenge is always to build a model that captures the actual statistical structure of the source as accurately as possible, because the model determines the code. This is why domain-specific codecs outperform general-purpose compressors: a JPEG encoder exploits knowledge of how natural images look (spatial correlations, dominant low spatial frequencies, human visual system properties) to build a model far more accurate than any general-purpose compressor could achieve for text.

The Kolmogorov-Chaitin-Solomonoff complexity of a string, which gives the length of the shortest program that outputs the string, is the ultimate theoretical limit of compression. Unlike Shannon entropy, which is a statistical concept averaging over a probability distribution, Kolmogorov complexity is defined for individual strings. A highly regular string like a billion repetitions of the digit 1 has very low Kolmogorov complexity despite having high Shannon entropy under the uniform distribution, because it can be generated by a short program. This distinction between typical strings (which are incompressible in both senses) and highly structured strings (which may be statistically rare but algorithmically simple) highlights the different perspectives that statistical and algorithmic information theory bring to compression.
